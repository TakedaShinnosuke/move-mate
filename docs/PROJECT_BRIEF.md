# move-mate プロジェクト計画書

健康維持アプリ「move-mate」の共同開発計画書です。開発開始前の合意形成と、開発中の参照ドキュメントとして使います。

本ドキュメントの目的は次の3点です。

1. プロジェクトの全体像を共有する
2. 担当分担とスコープを明確にする
3. 開発開始前に確認すべき事項を整理する

> 本バージョン (v0.3) は、v0.2 で合意した内容に対し、実際の Expo プロジェクト構造に合わせてフォルダパスを `src/` プレフィックス付きに更新した版です。

---

## 1. プロジェクト概要

「健康維持」を共通テーマに、**筋トレ**と**散歩**の両面から日常の運動習慣をサポートするスマートフォンアプリです。1つのアプリ内で2つの機能を分担して開発します。

- **筋トレ計測・AI提案機能**: 重量・回数の記録、過去履歴に基づく次回セットの提案
- **散歩ルート提案・週次ミッション機能**: ユーザー属性に応じたルート提案、ダイエット目的の達成可視化

## 2. 開発の目的

- 就職活動向けの成果物として、フルスタック開発・チーム開発の経験を積む
- 実際に自分たちが使う「ドッグフーディング」可能なアプリを作る
- モダンな技術スタック(React Native + Supabase + AI API)を実務に近い形で扱う
- Git/GitHub を用いた本格的なチーム開発フロー(PR レビュー、Issue 管理)を経験する

## 4. MVP スコープ

### 4.1 筋トレ機能([あなた] 担当)

1. 種目管理(BIG3 + 主要種目をシードデータとして初期登録)
2. セット記録(種目選択、重量・回数・セット入力)
3. 履歴閲覧(日付別の過去ワークアウト表示)
4. AI提案(種目の過去履歴を Claude API に渡し、次回推奨セットを生成)

### 4.2 散歩機能([友人] 担当)

1. 現在地取得(位置情報パーミッション)
2. 散歩時間または距離の入力
3. **ユーザー属性(性別・年齢)を考慮したルート提案**
   - 想定活用: 推奨ペース、推奨距離、消費カロリー目安の算出
4. 地図上にルート提案・表示
5. 散歩履歴の記録(経路と時間)
6. **週次ミッション機能(ダイエット目的)**
   - 事前定義のミッション(例: 「今週累計5km」「週3回散歩」など3〜5パターン)から週初めに自動選択
   - 進捗バー表示
   - 達成判定
   - 週次で自動リセット

### 4.3 共通機能

1. Google ログイン / ログアウト
2. タブナビゲーション(筋トレ / 散歩 / 設定)
3. ユーザープロフィール(氏名・性別・年齢・身長・体重)

### 4.4 v2 以降(MVP では作らない)

グラフ可視化、PR(自己ベスト)自動検出、プッシュ通知、フレンド機能、写真記録、ワークアウトテンプレート、散歩ルート共有、SNS連携、動的ミッション生成、複数同時ミッション、タブレット最適化 など。

## 5. 技術スタック

| 領域             | 採用技術                              | 選定理由                                                                  |
| ---------------- | ------------------------------------- | ------------------------------------------------------------------------- |
| フロントエンド   | React Native + Expo Router            | クロスプラットフォーム、JS/TS活用、Windows でも iOS ビルド可能(EAS Build) |
| 言語             | TypeScript                            | 型安全性、保守性、業界標準                                                |
| バックエンド・DB | Supabase(PostgreSQL + Auth + Storage) | 無料枠で十分、認証・DB・ストレージが統合                                  |
| 認証             | Google OAuth(Supabase Auth経由)       | UX良好、メール認証不要                                                    |
| 地図             | Google Maps Platform                  | ドキュメント充実、react-native-maps で扱える                              |
| AI               | Anthropic Claude API                  | 筋トレ提案の高品質生成、JS SDK あり                                       |
| バージョン管理   | Git + GitHub                          | 業界標準                                                                  |
| 開発支援         | Claude Code                           | 両者で利用前提                                                            |
| エディタ         | VS Code                               | 拡張機能が豊富、WSL / Mac 両対応                                          |

## 6. リポジトリ・開発運用

### 6.1 リポジトリ構成(モノレポ)

1リポジトリで両機能を管理。Expo Router 標準に従い、ソースコードは `src/` 配下に配置。機能ごとにフォルダ分離してコンフリクトを最小化。

```
src/
  app/
    (tabs)/
      strength/        # 筋トレ機能の画面
      walk/            # 散歩機能の画面
      home.tsx         # 共通ホーム画面
      profile.tsx      # プロフィール画面
  features/
    strength/          # 筋トレ機能のロジック
    walk/              # 散歩機能のロジック
  components/          # 共通UIコンポーネント
  lib/
    supabase.ts        # Supabase クライアント
    ai.ts              # Claude API クライアント
docs/
  PROJECT_BRIEF.md     # 本ドキュメント
```

### 6.2 ブランチ運用(GitHub Flow)

- `main` ブランチは常にデプロイ可能な状態を維持
- 作業は機能ブランチで実施(例: `feature/strength-set-input`、`feature/walk-route-display`)
- 完了後は Pull Request を作成 → もう一方がレビュー → マージ
- `main` への直接 push は禁止(Branch Protection Rule で強制)

### 6.3 コミット規約(Conventional Commits)

形式: `type(scope): subject`

| type     | 用途             |
| -------- | ---------------- |
| feat     | 新機能           |
| fix      | バグ修正         |
| refactor | リファクタリング |
| docs     | ドキュメント     |
| chore    | 設定・依存更新   |
| test     | テスト           |

例:

- `feat(strength): add set input form`
- `fix(walk): correct route distance calculation`
- `docs: update README with setup instructions`

### 6.4 タスク管理

GitHub Issues + Projects(カンバン形式)。

- 機能ごとに Issue を立てる
- ボード: `Todo / In Progress / Review / Done`
- ラベル: `strength`, `walk`, `common`, `infra`, `bug`, `enhancement`
